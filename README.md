# Active-Directory-Lab

# OverView
Active Directory home lab built with Windows Server and Windows 10/11 Enterprise using Oracle VirtualBox.

# Objectives 
- Install Windows Server
- Configure Active Directory Domain Services (AD DS)
- Promote the server to a Domain Controller
- Create Organizational Units (OUs)
- Create domain users
- Configure DNS
- Join Windows Client machines to the domain
- Verify domain authentication

  # Environment
  - Oracle VirtualBox
  - Windows Server
  - Windows 10 Enterprise Evaluation
  - Windows 11 Enterprise
  - Active Directory Domain Services
  - DNS
  - Virtual Networking

# Lab Steps
1. Created a Windows Server VM
2. Installed Active Directory Domain Services
3. Promoted server to Domain Controller
4. Created Organizational Units
5. Created domain users
6. Configured DNS
7. Joined the Windows client to the domain
8. Logged in using a domain account

# Lessons Learned
- Resource allocation is important when running multiple VMs
- Active Directory relies on proper DNS configuration.
- Domain clients must point to the Domain Controller for DNS.
- Virtual networking is critical for communication between VMs.
